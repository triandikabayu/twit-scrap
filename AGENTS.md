# AGENTS.md ‚Äî CF Twit Scrap

## Stack
- Python 3.x, FastAPI + Uvicorn, SQLite (WAL), Playwright, Jinja2, twikit (heavily monkey-patched)

## Setup

```
pip install -r requirements.txt
playwright install chromium
```

## Dev Commands

| Command | Action |
|---|---|
| `python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload` or just `python app/main.py` | FastAPI dev server at `0.0.0.0:8000` (auto-reload) |
| `python scripts/scraper.py --cookies cookies.json` | Headless scrape via Playwright |
| `python scripts/get_cookies.py` | Open browser for manual X login ‚Üí saves cookies |
| `python scripts/get_cookies.py --output cookies_N.json` | Per-account cookies |
| `python scripts/analyze.py` | Generate charts to `output/` from CSV |

## Scraping Flow
1. `python scripts/get_cookies.py` ‚Äî log in to X in the opened browser, close when home appears
2. Add keywords via web UI at `/keywords` (priority keywords cross-product with non-priority)
3. POST `/api/scrape/start` ‚Äî runs `scripts/scraper.py` as subprocess, streams via SSE at `/api/scrape/stream`
4. Data lands in `data/comifuro_tweets.csv` + `.json`, then imported into SQLite

## Critical Architecture
- **`app/main.py`** ‚Äî FastAPI entrypoint; imports all route modules and includes their routers
- **`app/routes/scrape.py`** ‚Äî spawns `scripts/scraper.py` as subprocess, parses `TWEET_JSON:{...}` stdout lines, streams via SSE. Contains shared SSE state
- **`app/helpers.py`** ‚Äî shared helpers (`_session_required`, `_render`, `_PRIORITY_RE`, etc.)
- **`app/models/db.py`** ‚Äî SQLite at `data/cf_scraper.db`, WAL mode, session-based multi-tenant, schema auto-created on app startup
- **`scripts/scraper.py`** ‚Äî Playwright on `x.com/search`, uses `data-testid` selectors (brittle), `SCROLL_LIMIT=50`, scroll delay `(3,5)`, date range hardcoded in both `config/settings.py` and `scripts/scraper.py`
- **`config/settings.py`** ‚Äî central config (creds, cookies file, date range, `SESSION_EXPIRY_MINUTES=30`); **contains plaintext Twitter credentials ‚Äî do not commit**
- **`app/data/characters.py`** ‚Äî ~40+ fandoms hardcoded as `KNOWN_CHARACTERS`, imported as module by routes and db
- **`app/data/fandoms.py`** ‚Äî local character DB per fandom with Wikipedia API fallback

## twikit Monkey Patches
`keys.py` contains all patches for `twikit` 2.x to work with current X.com. These break when X updates its JS bundles:
- `ON_DEMAND_FILE_REGEX` ‚Äî regex for ondemand JS bundle index
- `ClientTransaction.get_indices` ‚Äî different ondemand URL pattern
- `GQLClient.search_timeline` ‚Äî GET ‚Üí POST
- `User.__init__` ‚Äî handles missing entity keys
Call `apply_all_patches()` to apply all at once.

## Quirks
- No linting, typechecking, or test infrastructure
- `keywords.txt` is auto-generated by API from DB (cross-product of priority √ó non-priority), but can also be hand-edited
- Cookie files (`cookies.json`, `cookies_{id}.json`) live in project root ‚Äî not gitignored (verify `.gitignore`)
- Session expiry: 30 min from `config/settings.py`
- Priority = 0/1 binary flag on keywords
- FastAPI docs at `/docs` is disabled (`docs_url=None`)
- The scraper page now has keyword checkboxes ‚Äî only selected keywords are scraped
